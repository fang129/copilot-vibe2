## 2. 19MC改-紧急通报数据样例
### 请求Body
```
REQ_TYP=00&C_TYP=0&C_NO=02&C_TIM=2007-03-01T14%3a23%3a54%2b00%3a00&U_TEL=13911481234&U_ID=8675150010123456&S_KIND=10000000&P_TYP=01&P_NO=03&P_TIM3=2007-03-01T09%3a57%3a35%2b00%3a00&P_LAT1=N4785f5&P_LON1=E11769eb&P_DIR3=346&P_SPD3=51&P_DEF3=99999&P_TIM2=2007-03-01T09%3a57%3a39%2b00%3a00&P_LAT2=N478635&P_LON2=E11769d4&P_DIR2=342&P_SPD2=51&P_DEF2=99999&P_TIM1=2007-03-01T09%3a57%3a43%2b00%3a00&P_LAT3=N478675&P_LON3=E11769c3&P_DIR1=342&P_SPD1=52&P_DEF1=99999
```

### 参数说明
- **P_TYP**：测地系信息
  - 第1位：0=02中国测地系；1=84世界测地系
  - 第2位：1=度单位（16进制）1/（2^17）°；2=度单位（10进制）
- **P_NO**：坐标个数
- **P_LATn**：纬度（n为坐标序号）
  - 第1位：N=南纬；1位后为纬度值
- **P_LONn**：经度（n为坐标序号）
  - 第1位：E=东经；1位后为经度值

## 4. 19MC改-紧急通报处理流程
### 测地系信息抽出规则
- 仅抽出：**01、11、02、12**；除此以外不抽出（*1）

| No. | 项目名 | 取值说明 |
| ---- | ---- | ---- |
| 1 | 坐标情报测地系（第1位） | 0=02中国测地系；1=84世界测地系 |
| 2 | 经纬度单位（第2位） | 1=度单位（16进制）；2=度单位（10进制） |

### DCM数据结构
| 数据类型 | 项目名 | 说明 |
| ---- | ---- | ---- |
| 测地系信息 | P_TYP | 1位测地系+1位经纬度单位 |
| 坐标信息 | P_LAT1/P_LON1、P_LAT2/P_LON2… | 纬度/经度序列 |
| 坐标个数 | P_NO | 坐标数量 |

### 坐标筛选规则
1. **有效坐标抽出**：经纬度都非空且有效
2. **空/无效坐标抽出**：经纬度任一为空

### 无效化处理流程
1. 测地系替换：84系→02中国测地系（11→01、12→02）（*2）
2. 所有坐标统一替换为**无效坐标**
3. 发送至ECDP；送信失败返回**HTTP500**（*3）

### Pattern说明
- **B Pattern**：流用19MC原有逻辑
- **D Pattern**：直接透传给ECDP系统

### 注释
- *1：测地系非01/11/02/12时，执行无效化处理
- *2：仅84坐标系替换为02；其他测地系保持不变
- *3：送信失败→DCM返回HTTP500

## 5. 19MC改-紧急通报PatternD处理概述
### 处理概要
19MC改紧急通报核心为**请求透传**至下游ECDP系统，步骤：
1. 接收车认请求，删除部分Header
2. Content-Type：`application/octet-stream` → `application/x-www-form-urlencoded`
3. 提取Body二进制数据，转发至ECDP
4. 接收ECDP应答，Header Key首字母大写后返回车认

### Header处理规则
#### 送信前删除
Content-Encoding、Accept-Encoding、Content-Length、Host、Proxy-Connection、Cache-Control、Connection

#### 送信前替换
Content-Type → `application/x-www-form-urlencoded`

#### 应答Header转换（小写→大写）
- content-type → Content-Type
- user-agent → User-Agent
- x-d-authenticate → X-D-Authenticate
- accept-language → Accept-Language
- content-length → Content-Length
- accept → Accept

### 异常处理
送信失败时，向DCM返回**HTTP500**
