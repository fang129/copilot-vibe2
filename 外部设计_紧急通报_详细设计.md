## 3. 功能概要
### 3.1 业务功能
| No | 业务功能 | 处理概要 | 备注 |
|----|----------|----------|------|
| 1  | 紧急通报 | 发生紧急情况时，向紧急呼叫中心进行紧急呼叫 | - |

### 3.2 功能一览
| No | 功能名 | 处理概要 | 备注 |
|----|--------|----------|------|
| 1  | 坐标变换处理 | B pattern：解析二进制电文→结合数据样例提取坐标→坐标无效化→转发给下游Center<br>D pattern：二进制电文直接透传下游Center | - |

### 3.3 处理流程
```
开始
→ 取得接收数据
→ Pattern判定（B/D）
   ├─ D pattern → 直接HTTP送信
   └─ B pattern → 坐标值抽出 → 无效值处理 → 坐标值替换 → 送信处理
结束
```

## 4. 坐标值变换功能
### 4.1 处理说明
#### 数据样例（请求Body）
```
REQ_TYP=00&C_TYP=0&C_NO=02&C_TIM=2007-03-01T14%3a23%3a54%2b00%3a00&U_TEL=13911481234&U_ID=8675150010123456&S_KIND=10000000&P_TYP=01&P_NO=03&P_TIM3=2007-03-01T09%3a57%3a35%2b00%3a00&P_LAT3=N4785f5&P_LON3=E11769eb&P_DIR3=346&P_SPD3=51&P_DEF3=99999&P_TIM2=2007-03-01T09%3a57%3a39%2b00%3a00&P_LAT2=N478635&P_LON2=E11769d4&P_DIR2=342&P_SPD2=51&P_DEF2=99999&P_TIM1=2007-03-01T09%3a57%3a43%2b00%3a00&P_LAT1=N478675&P_LON1=E11769c3&P_DIR1=342&P_SPD1=52&P_DEF1=99999
```

#### 参数说明
- `P_TYP`：测地系信息
- `P_LATn`：纬度（n为坐标序号）
- `P_LONn`：经度（n为坐标序号）

---

### 4.1.1 取得接收数据
| 判定条件 | 处理 |
|----------|------|
| 变换 pattern = D | 直接进入HTTP送信处理 |
| 请求数据 = NULL | 不做坐标变换，直接进入HTTP送信处理 |
| 其他 | 继续坐标变换流程 |

---

### 4.1.2 坐标值抽出
#### (1) 测地系信息（P_TYP）校验
- 由请求内容中获取P_TYP元素，按下表进行输入校验。（P_TYP是记录位置信息所适用测地系以及数据格式的元素）
- 格式：2位数字
  - 第1位：0=中国测地系；1=WGS84测地系
  - 第2位：1=16进制度单位1/（2^17）°；2=10进制度单位
- 有效范围：`01、02、11、12`

| 判定条件 | 处理 |
|----------|------|
| P_TYP为空 或者 不在有效范围 | 输出错误日志（无法解析报文坐标信息（测地系不正）`E-{0}-106`，继续坐标抽出并做**坐标无效值处理** |
| 校验通过 | 执行坐标变换 |

#### (2) 坐标抽出规则
- 从请求参数中抽出所有位置信息，提取所有`P_LAT`/`P_LON`开头的位置信息
- 同序号`P_LATn`+`P_LONn`为一组位置
- 经/纬度任一为空/无效 → 该组**执行坐标无效值处理**
- 经/纬度均有效 → 该组**执行坐标无效值处理**

---

### 4.1.3 坐标无效值处理
按进制用环境变量替换无效坐标：
| P_TYP第2位 | 纬度 | 经度 |
|------------|------|------|
| 1（16进制） | `INVALID_COORDINATE_HEX` | `INVALID_COORDINATE_HEX` |
| 2（10进制） | `INVALID_COORDINATE_DEC` | `INVALID_COORDINATE_DEC` |

---

### 4.1.4 坐标值替换
1. 位置信息：用变换结果替换原`P_LATn`/`P_LONn`
2. 测地系：`P_TYP`第1位=1 → 强制改为0（中国测地系）

---

### 4.1.5 送信处理
执行**HTTP送信功能**

## 5. HTTP送信功能
### 5.1 处理说明
1. **HTTP请求设定**
   | 通信项目 | 设定项 | 设定规则 |
   |----------|--------|----------|
   | URI | 转发目标URI | 19MC改车认服务配置 |
   | Header | 请求头 | 透传并清理/替换指定字段 |
   | Method | 请求方法 | 透传原车认服务Method |
   | BODY | 请求体 | 透传原车认服务数据 |

2. **Header处理规则**
   - 删除：`Content-Encoding`/`Accept-Encoding`/`Content-Length`/`Host`/`Proxy-Connection`/`Cache-Control`/`Connection`
   - 替换：`Content-Type: application/octet-stream` → `application/x-www-form-urlencoded`

3. **通信组件**
   - 使用Spring `RestTemplate` 发送请求

4. **通信结果判定**
   | 判定条件 | 处理 |
   |----------|------|
   | HTTP Status = 200 | 输出`I-{}-0001`日志，透传响应给上游 |
   | HTTP Status ≠ 200 | 输出`E-{0}-0301`日志，透传响应 |
   | 超时未响应 | 按配置重试，输出`E-{0}-0302`日志，进入异常处理 |
   | 超过最大重试次数 | 输出`E-{0}-0303`日志，进入异常处理 |
   | 其他异常 | 输出`E-{0}-0399`日志，进入异常处理 |

5. **响应Header大小写转换**
   | 原字段 | 转换后 |
   |--------|--------|
   | content-type | Content-Type |
   | user-agent | User-Agent |
   | x-d-authenticate | X-D-Authenticate |
   | accept-language | Accept-Language |
   | content-length | Content-Length |
   | accept | Accept |

---

## 6. 异常处理
- 统一返回：**HTTP 500**
- 响应Body：**空**

## 7. 内部/共通处理详细
### 7.1 日志定义（业务代码：MCMOD_EMGCA）
| 日志编码 | 日志消息 | 说明 |
|----------|----------|------|
| I-{0}-0001 | 处理正常终了 | 正常完成 |
| E-{0}-0001 | 动作定义文件配置错误 | 配置异常 |
| E-{0}-0002 | 环境配置信息错误 | 环境变量异常 |
| E-{0}-0301 | 接口调用失败（响应异常） | 下游返回非200 |
| W-{0}-0302 | 接口调用失败（响应超时） | 请求超时 |
| E-{0}-0303 | 接口调用失败（重试超次） | 超过最大重试 |
| E-{0}-0399 | 接口调用失败（意外错误） | 未知异常 |

### 7.2 环境变量
| No | 设定项目 | 属性名 | 默认值 |
|----|----------|--------|--------|
| 1 | 十六进制无效坐标 | INVALID_COORDINATE_HEX | FFFFFFFF |
| 2 | 十进制无效坐标 | INVALID_COORDINATE_DEC | 00.0000000 |

### 7.3 动作定义文件
| No | 属性名 | 设定项目 | 说明 |
|----|----------|--------|--------|
| 1 | Behavior | - | 数组 |
| 2 | PatternURL | 对象URL | 匹配业务入口 |
| 3 | RewriteUrl | 转发URL | 下游目标地址 |
| 4 | Timeout | 超时时间 | 单位：ms |
| 5 | RetryCount | 重试次数 | 超时重试次数 |
| 6 | IntervalBetweenRetries | 重试间隔 | 单位：ms |
| 7 | Pattern | 变换Pattern | B/D模式区分 |
